## 其他
在AI编程盛行的今天，老一辈艺术家还在坚持手搓代码（

给自己的医疗小模型搭了一个对话页面，不过里面用的是微调前的base模型所以有些胡言乱语

虽然说是手搓，但其实生成式代码也占到了至少一半，我觉得这是利用AI辅助学习最大需要去思考的点🤔：如何在AI接管和上手学习之间找到那个最佳的平衡点：

如果我只管发指令，那大概率一天之内就能搞完，如果我自己写所有代码而只向AI请教更宏观的设计问题，那我这双刚接触nextjs的生疏的手估计得写个半个月；

所以我基本上采取如下模式：

- UI样式纯让AI生成，不用管

- 第一次接触的模式和概念先让AI写到能运行，然后自己再一句句地理解并且后续模仿自己写

- 多和AI讨论代码组织和不同模式的问题，因为会牵涉到框架背后的核心能力与开发模式，然后根据这些概念自己重新组织整理代码，并和AI重新对齐（一些RLAIF哈哈哈）

以前写代码是先理解后编程，现在主打一个先婚后爱，先让AI编程跑通再开始理解🤣
